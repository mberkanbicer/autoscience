④ Data Flow Patterns

🧵 Primary Research Flow

User submits prompt → API creates idea + run → Orchestrator builds workflow → Workflow iterates 16 steps → Each step updates ResearchState in-memory → PersistenceService writes to DB → Report/Wiki generated → Events broadcast via Redis Pub/Sub → Frontend receives SSE updates

File: orchestrator.py → research_workflow.py → research_persistence_service.py

⏰ Idle Cognition Flow

IdleScheduler (APScheduler, every 5 min) → checks user inactivity + budget → selects idle mode (frontier scan / citation-conflict / revisited rejected / cross-domain / skill improvement / dataset discovery) → IdleCognitionEngine runs → results stored → wiki updated

File: idle_scheduler.py → idle_cognition.py

🔄 State Persistence Flow

ResearchState (Pydantic, in-memory) → ResearchPersistenceService.persist_run_results() → batch dedup papers → store papers/clusters/conflicts/questions/hypotheses → embed new papers → update run metrics → flush

File: research_persistence_service.py → deduplication.py → embedding_service.py

📡 Real-time Event Flow

Workflow step executes → _broadcast_event() called → 1) persist to DB via run_service.add_run_event() → 2) publish to Redis channel run:{run_id}:events via EventBroadcaster → Frontend SSE endpoint subscribes → events sent as text/event-stream

File: research_workflow.py → event_stream.py → sse_stream.py

🔒 Safety Gate Flow

Before each gated step → enforce_step_safety_gate() checks: 1) is action permitted? 2) will projected cost exceed $1 threshold? 3) budget remaining? → if blocked, creates ApprovalRequest in DB → raises ApprovalGateError → workflow pauses → frontend shows DecisionGate component

File: safety_gates.py → safety_service.py

🔄 Snapshot/Restore Flow

Resume API called → SnapshotService.create_snapshot(run_id) → loads Run + Idea + Papers + Clusters + Conflicts + Questions + Hypotheses + Scores + Events + ToolCalls from DB → reconstructs full ResearchState → workflow.run_from_step(state, start_step)

File: snapshot_service.py
